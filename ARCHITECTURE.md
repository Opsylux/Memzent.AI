Aura: System Architecture

This document defines the high-level data flow and service boundaries for Aura, an enterprise-grade AI Gateway and Semantic Router.
1. High-Level Data FlowThe system operates as an "Intelligent Proxy" between a user client and Large Language Models (LLMs).Ingress: The Go Gateway receives an HTTP request containing a user prompt.Semantic Cache Check: The Gateway checks Valkey (using a hash of the prompt) to see if a routing decision has already been made.Vector Routing: If no cache exists, the Gateway calls the Rust Router via gRPC.The Rust Router converts the prompt into a vector (Phase 2).It queries Qdrant for the top $k$ most relevant tools.Governance & RBAC: The Gateway cross-references the selected tools against Postgres to ensure the user_id has permission to use them.LLM Execution: The Gateway sends the final, pruned prompt to the LLM and returns the response to the user.2.
